# 予約管理スプレッドシート 設計書（商品版）

予約システムのデータ基盤となる Google スプレッドシートの設計。
ポイントは **「予約状況がカレンダー形式（日 × 時間で 空き / 仮 / 確定）で一元管理でき、かつ予約者名・連絡先も紐づく」** こと。

従来は「状況シート（状態のみ）」と「問い合わせシート（顧客情報）」が分離しており、状況側に予約者名・連絡先が無かった。
商品版では両者を **予約ID** で連動させ、状況カレンダーのマスからその予約の顧客情報へたどれるようにする。

> 本書は汎用設計のみを記載し、特定の事業者・個人・サービス名などの固有情報は含めない。

---

## 全体構成（シート一覧）

| シート名 | 役割 | 主キー |
|---|---|---|
| `予約状況` | 日 × 時間のマトリクスで空き状況を可視化・一元管理 | 日付 × 時間 |
| `予約明細` | 1予約 = 1行。顧客情報・希望日時・状態・メモを保持 | 予約ID |
| `履歴` | 状態変更のログ（任意・運用補助） | タイムスタンプ |
| `設定` | 営業時間・受付ルール等（任意） | キー |

`予約状況`（カレンダー）と `予約明細`（顧客情報）を **予約ID** で結ぶのが本設計の中心。

---

## 1. 予約状況カレンダー（シート名: `予約状況`）

日付を行、時間帯を列に取った **マトリクス表**。1マス = ある日のある時間枠の状態。
写真館などで実運用されている「日 × 時間枠の一覧表」形式を踏襲する。

### レイアウト

| 列 | 内容 | 例 |
|---|---|---|
| A | 日付 | `2026-08-16` |
| B | 曜日 | `Sat` |
| C | コメント / 終日操作 | `定休日` / `仮受OK`（行全体の一括操作にも使う） |
| D 以降 | 時間枠（30分刻み等） | `09:00` `09:30` `10:00` … `18:30` |

- 1行 = 1日。D列以降のヘッダ行（1行目）に時間ラベルを置く。
- 各時間マスに「状態」と「予約者名（短縮）」を表示する。
  - 表示例: `確定 山田` / `仮 佐藤` / 空きは空欄
- C列（コメント）は「定休日」「臨時休業」などの終日メモや、行全体への一括設定に使う。

### マスが取りうる状態

| 状態 | 意味 | 色（背景）方針 |
|---|---|---|
| （空欄） | 空き | 着色なし（白） |
| `仮予約` | お客様が申し込み、店主未確認 | 黄系（例 `#FBF0D2`） |
| `仮受OK` | 店主が受付OKを返答（確定前の内諾） | 薄緑系（例 `#E4F1E5`） |
| `確定` | 店主が予約を確定した状態 | 緑系・濃いめ（例 `#C8E6C9`） |
| `キャンセル` | 取消（記録として残す場合） | グレー系（例 `#ECECEC`） |
| `NG` / `×` | 受付不可（手動ブロック） | グレー／斜線 |
| `定休日` | 終日休業 | グレー（C列に表示） |

> 「状態のみ」ではなく **「状態 + 予約者名」** をマスに持たせることで、表を見ただけで「いつ・誰の予約が入っているか」が分かる。これが従来の状況シートとの最大の違い。

### 色分け方針

- 空き＝白、仮系＝黄〜薄緑、確定＝緑、不可＝グレー、と「緑に近づくほど確実」というグラデーションで直感的に。
- 条件付き書式（セルの値が `確定` なら緑、`仮予約` なら黄…）で自動着色。
- 名前はマス内の小さめ文字で添える（`確定\n山田 様` のような2段表示でも可）。

## 2. 予約明細（シート名: `予約明細`）

1予約 = 1行。フォーム送信時に自動追記され、顧客情報・希望日時・状態をすべて保持する。

### 列定義

| 列 | 列名 | 型 | 意味 |
|---|---|---|---|
| A | 予約ID | 文字列 | 予約の一意キー。例 `R-20260801-001`。状況カレンダーのマスから参照する鍵 |
| B | 受付日時 | 日時 | フォーム送信を受け付けた日時（自動） |
| C | 希望日時 | 日時 / 文字列 | お客様が選んだ希望日時。`2026-08-16 10:00` 形式。未定の場合は「未定」 |
| D | お名前 | 文字列 | 予約者氏名 |
| E | メールアドレス | 文字列 | 連絡先メール（お客様通知の宛先） |
| F | 電話番号 | 文字列 | 連絡先電話（任意） |
| G | サービス種別 | 文字列 | 申し込まれたメニュー / コース / レッスン種別など |
| H | オプション | 文字列 | 業種別の追加項目（指名スタッフ / レベル / 訪問エリア 等） |
| I | 状態 | 文字列 | `未対応` → `仮受OK` → `確定` → `完了` / `キャンセル` |
| J | 店主メモ | 文字列 | 内部用の自由記入（折り返し対応・備考など） |
| K | カレンダーイベントID | 文字列 | Google カレンダー連携時のイベントID（自動） |
| L | 通知ログ | 文字列 | 送信した通知の記録（受付確認 / リマインド送信済 等。任意） |

### 状態の遷移

```
未対応 ──(店主が受付OK)──▶ 仮受OK ──(店主が確定)──▶ 確定 ──(当日後)──▶ 完了
   │                          │                         │
   └──────────── キャンセル ◀──┴─────────────────────────┘
```

- 自動確定にはしない。**店主が「承認 / 確定」を選べる**運用（仮受 → 確定の2段階）。
- 状態列の値は `予約状況` カレンダーのマス色・表示と対応させる。

## 3. 連動仕様（カレンダー ⇄ 明細）

### 3-1. フォーム送信時（新規予約）

1. `予約明細` に1行追記
   - 予約ID を採番（例 `R-{受付日YYYYMMDD}-{連番3桁}`）
   - 受付日時・希望日時・お名前・連絡先・サービス種別・オプションを記録
   - 状態 = `未対応`（または `仮予約`）
2. `予約状況` カレンダーの該当マス（希望日の行 × 希望時間の列）に
   - 状態 `仮予約` と 予約者名（短縮）を書き込む
   - 同時に予約IDをマスのメモ（ノート）または隣接の非表示列に保持し、明細へたどれるようにする
3. 店主へ通知（LINE / メール）、お客様へ受付確認メールを送信

### 3-2. 承認 / 確定 / キャンセル時（状態更新）

| 操作 | `予約明細`.状態 | `予約状況` マス |
|---|---|---|
| 受付OK（内諾） | `仮受OK` | `仮受OK 〇〇`（薄緑） |
| 確定 | `確定` | `確定 〇〇`（緑） |
| キャンセル | `キャンセル` | 空欄に戻す or `キャンセル`（グレー） |
| 完了（当日後） | `完了` | （任意で淡色化） |

- 状態はカレンダー側のマス編集・明細側の状態列編集・操作画面のいずれからでも更新でき、もう一方に反映する（双方向同期）。
- 確定時に Google カレンダーへイベントを作成し、そのイベントIDを `予約明細`.K に保持。

### 3-3. マスから明細への参照方法

- 各マスにひも付けた **予約ID** をキーに `予約明細` を引く。
  - 実装例: マスのノートに `予約ID=R-20260801-001` を記録／別の非表示シートで「日付・時間 → 予約ID」を対応付け。
- これにより「カレンダーで気になるマスを見る → その予約の名前・連絡先・メモを即確認」が1クリックで完結する。
- 逆に `予約明細` の希望日時から、`予約状況` の対応マスを特定して色・表示を更新する。

### 3-4. 設計上の注意

- 1つの時間枠に複数予約が入りうる業種では、マスに代表1件＋件数（例 `確定 山田 ほか1`）を表示し、詳細は明細で見る。
- 予約者名はカレンダー上では氏のみ等の短縮表示にし、フルの個人情報は `予約明細` に集約（表示範囲を絞れる）。
- 予約ID をキーにすることで、氏名重複・改名・連絡先変更があっても1予約を一意に追跡できる。

---

## まとめ

| 観点 | 従来（分離型） | 商品版（連動型） |
|---|---|---|
| 状況の見え方 | 状態のみ（誰の予約か不明） | 状態 + 予約者名がマスに出る |
| 顧客情報 | 別シートに分離 | 予約IDで明細へ即参照 |
| 一元管理 | 2シートを突き合わせる必要 | カレンダー1枚から全情報へ到達 |

**カレンダー形式で予約状況を一元管理しつつ、予約者名・連絡先も紐づく** —— これが商品の中核価値であり、本設計で実現する。
